1. Типы листов стилей



CSS (Cascading Style Sheets) — язык таблиц стилей, который позволяет прикреплять стиль (например, шрифты и цвет) к структурированным документам (например, документам HTML и приложениям XML). Обычно CSS-стили используются для создания и изменения стиля элементов веб-страниц и пользовательских интерфейсов, написанных на языках HTML и XHTML. Отделяя стиль представления документов от содержимого документов, CSS упрощает создание веб-страниц и обслуживание сайтов.

Чтоб создать CSS-файл, создайте файл блокнота и переменуйте его, изменив расширение на .css

Существует четыре способа применения листов стилей в документе HTML:


 1 вариант. Внутри открывающего тега с помощью атрибута style.

 Элементам на странице, которые располагаются внутри элемента body, можно добавить атрибут style. Значением этого атрибута могут быть свойства и значения CSS, которые должны будут применены к этому элементу.

 Давайте посмотрим, как это работает на конкретном примере. Атрибут style добавлен для элемента <p>.

<!DOCTYPE HTML>
<html>
<head>
<title>Документ без названия</title>
</head>
<body>
<p style="color:red;">Абзац</p>
</body>
</html>

 Обратите внимание, что в этом случае не нужно использовать селектор, т.к. элемент, к которому добавлены стили уже определен.

 2 вариант. Внутри элемента style.

 Еще один способ подключения стилей CSS, это воспользоваться элементом <style>.

 Давайте посмотрим, как это выглядит на конкретном примере.

<!DOCTYPE HTML>
<html>
<head>
<style>
p {color:red;}
</style>
<title>Документ без названия</title>
</head>
<body>
<p>Абзац</p>
</body>
</html>

 3 вариант. Подключение внешнего файла стилей.

 3 вариант подключения стилей CSS является использование элемента link, который позволяет подключать к HTML-странице внешние файлы.

 Обратите внимание на атрибут, который указывается у этого элемента.

<link rel="stylesheet" href="style.css" >

 Он тоже является обязательным. В атрибуте href указывается путь к css-файлу, который нужно подключить.

<!DOCTYPE HTML>
<html>
<head>
<link rel="stylesheet" href="style.css">
<title>Документ без названия</title>
</head>
<body>
<p>Абзац</p>
</body>
</html>

 4 вариант. Использование импорта.

Используя ключевое слово @import, можно импортировать лист стилей в HTML-файл. Не забудьте, что вы просто импортируете текстовый файл, а потому вы должны вставить его в контейнер <style>. В этом случае импортирование листа стилей происходит точно так же, как внедрение листа в HTML-файл. Например:

<style> 
  @import url(http://www.myserver.coin/style.css);
</style>

 В современной верстке веб-страниц, невозможно представить создание сайта без CSS, поэтому советую вам поэкспериментировать с подключением CSS к своей веб-странице. В будущем, это вам очень сильно пригодится.


2. Синтаксис листов стилей


Каскадные таблицы стилей описывают правила форматирования элементов с помощью свойств и допустимых значений этих свойств. Для каждого элемента можно использовать ограниченный набор свойств, остальные свойства не будут оказывать на него никакого влияния.

Объявление стиля состоит из двух частей: селектора и объявления. В HTML имена элементов нечувствительны к регистру, поэтому «h1» работает так же, как и «H1». Объявление состоит из двух частей: имя свойства (например, color) и значение свойства (grey). Селектор сообщает браузеру, какой именно элемент форматировать, а в блоке объявления (код в фигурных скобках) перечисляются форматирующие команды — свойства и их значения.


Хотя приведенный пример пытается влиять только на пару свойств, необходимых для рендеринга HTML-документа, он сам по себе квалифицируется как таблица стилей. В сочетании с другими таблицами стилей (одна фундаментальная особенность CSS заключается в том, что таблицы стилей объединяются), правило будет определять окончательное представление документа.

h1 {
type-family: garamond, times, serif;
}

Обратите внимание на то, что объявление взято в фигурные скобки. Данное правило распространяется только на внедренные и связанные стили. При объявлении встроенного стиля используются кавычки. Кроме того, в этом случае не используются селекторы, так как стиль применяется только к дескриптору, в котором он определен:

<h1 style="type-family; garamond, times, serif;">Стиль текущего абзаца определен соответствующим объявлением.</h1>

Составные объявления разделяются точкой с запятой (;). В случае единичного объявления в точке с запятой нет необходимости.


3. Именование и объединение листов стилей


Атрибут title дескриптора <link> позволяет дать имя листу стилей. Если вы не указываете значение title, лист стилей является постоянным (persistent), и пользователь не может отключать его (если только он не отключает листы стилей в целом). Присвоение значения title изменяет тип листа стилей на стандартный (default), и пользователь может выбирать стили из числа именованных.
браузер объединяет любые листы стилей с одинаковыми заголовками, которые имеют ключевое слово alternate, присвоенное атрибуту REL.
В следующем примере браузер применяет стили как из файла first.css, так и из second.css, поскольку дескрипторы <link> присваивают атрибуту REL значение alternate и имеют один и тот же заголовок.

<link  REL="alternate stylesheet" title="Example"
HREF="first.css">
<link  REL="alternate stylesheet" title="Example1'
HREF="second.css">

Вы можете переназначить старшинство для правила, используя ключевое слово important. В следующем примере присваивание значения red свойству color и значения sans-serif свойству font-family отмечено как important. В этом случае браузер не будет отменять эти стили. Если два конкурирующих листа стилей отмечают одно и то же свойство как important, применяются приведенные ранее правила.

h1 (color:red ! important font-weight:bold font-family:sans-serif ! important}


4. Добавление правил к листу стилей


Связанные и вложенные листы стилей позволяют определять стили для одного или нескольких отдельных дескрипторов. Например, вы можете создать лист стилей, который определяет стили для дескрипторов <h1>, <h2>, <p>, <em>. Каждое определение стилей называется правилом (rule). Правило содержит селектор (selector), представляющий собой дескриптор HTML, сопровождаемый декларацией (описанием стиля). Селектор правила связывает определение стиля с дескрипторами, используемыми в HTML-файле. Вот пример правила, которое определяет стиль для каждого дескриптора <h1>: 

h1 {color  :  blue}

Объявление заключено в фигурные скобки {}. Каждый элемент в объявлении имеет две части: имя свойства и значение, которое вы присваиваете ему, разделенные двоеточием (:). В этом примере color - имя свойства, a blue - значение, которое вы ему присваиваете. HTML определяет множество имен свойств (размер шрифта, стиль шрифта, цвет, правое поле и так далее). Каждое свойство имеет также заранее определенный тип и диапазон значений.
Следующий пример показывает несколько правил, содержащихся внутри вложенного листа стилей:

<style >
h1 {color: blue}
P {font-size: 10pt} 
</style>

5. Установка нескольких свойств в одном правиле


Вы можете установить несколько свойств внутри одного правила. Это можно сделать, отделяя каждое присвоение значений точкой с запятой (;):

h1 {color: blue; font-size: 12pt; text-line: center}

В данном случае браузер будет выводить заголовки <h1> синим цветом, используя шрифт размером 12 пунктов, выравнивая текст в окне браузера по центру. Для всех других свойств браузер будет использовать значения по умолчанию. В данном случае свойство font-style будет равно normal.


6. Группирование селекторов в одном правиле


Если вы хотите определить один и тот же стиль для нескольких дескрипторов, можно вносить их в ваш лист стилей списком, как показано далее:

Р {font-size: 12pt} 
UL {font-size: 12pt} 
LI {font-size: 12pt}

Вы можете также группировать селекторы вместе и определять правило для них как для группы. В следующем примере группируются селекторы в одной строке и определяется правило, которое устанавливает свойство font-size, равным 12pt:

Р, UL, LI (font-size: 12pt}

Обратите внимание на запятые в списке между селекторами. Пропуск этой запятой приведет к совершенно другому результату.


7. Определение наследования в правилах


Используя листы стилей HTML, вы можете очень точно определить применение стиля к дескриптору. Например, вы можете определить два стиля для дескриптора <LI>: один для случая, когда он порожден дескриптором <UL> и другой, когда он порожден дескриптором <OL>. Это делается с помощью контекстных селекторов.

Контекстные селекторы определяют точную последовательность дескрипторов, для которых будет применяться стиль. Другими словами, можно определить, что стиль будет применяться к специфическому дескриптору типа <LI> только в том случае, когда он порожден дескриптором <OL>, как показано далее: 

OL LI {list-style-type: decimal}

Таким же образом можно определять, что некоторый стиль применяется к дескриптору <LI> только в том случае, когда он порожден дескриптором <UL>:

UL LI {list-style-type: square}

 Обратите внимание, что в этом случае список селекторов не разделен запятыми.


8. Работа с классами в листах стилей


Класс определяет разновидность стиля, к которому вы обращаетесь в определенном дескрипторе, используя атрибут CLASS. Например, можно определить три разновидности стиля h1 и затем использовать каждый из них в соответствующем контексте. Класс определяется во многом подобно тому, как обычно определяют стиль, с той разницей, что к дескриптору добавляется произвольное имя класса, отделяемое точкой, как в следующем примере: 

h1.blue {color: blue} 
h1.red {color: red} 
h1.black {color: black}

При добавлении дескриптора <h1> к вашему HTML-документу, вы устанавливаете атрибут CLASS, чтобы указать, какoй именно стиль вы намерены использовать:

<h1 CLASS=red>Kpacный заголовок</h1>


9. Определение стиля для специфического элемента


Вы можете использовать оперативный стиль для определения внешнего вида элемента, указывая его в атрибуте style. Вы можете также сделать ваш HTML-документ легко читаемым, присваивая элементу идентификатор (ID); затем вы ставите в соответствие стиль этому элементу, используя ID.
Например, следующий дескриптор <p> присваивает атрибуту абзаца ID значение test: 

<Р ID=test>Tecтовый абзац</p>

Теперь можно определить правило, которое действует только на данный элемент, используя префикс # и идентификатор в качестве селектора для правила: 

test (color: red)


10. Работа с псевдоклассами


Псевдоклассы определяют динамическое состояние элементов, которое изменяется с помощью действий пользователя, а также положение в дереве документа. Примером такого состояния служит текстовая ссылка, которая меняет свой цвет при наведении на неё курсора мыши. При использовании псевдоклассов браузер не перегружает текущий документ, поэтому с помощью псевдоклассов можно получить разные динамические эффекты на странице. Синтаксис применения псевдоклассов следующий.

  Селектор:Псевдокласс { Описание правил стиля }

  Первое, что нужно понять, это концепция состояний ссылок - различных состояний, в которых могут существовать ссылки, которые могут быть стилизованы с использованием различных псевдоклассов :

  Ссылка (не посещенная) : состояние по умолчанию, в котором находится ссылка, когда она не находится в каком-либо другом состоянии. Это может быть специально стилизовано с использованием :link псевдокласса.

  Посещенные : ссылка, когда она уже была посещена (существует в истории браузера), стилизованная с использованием :visited псевдокласса.

  Наведение : ссылка, когда на нее указывает указатель мыши, стилизованный с использованием :hover псевдокласса.

  Фокус : ссылка, когда она была сфокусирована (например, перемещена пользователем клавиатуры с помощью Tab клавиши или подобного, или программно сфокусирована с использованием HTMLElement.focus()) - это оформляется с использованием :focus псевдокласса.

  Активно : ссылка, когда она активируется (например, по нажатию), стилизована с использованием :active псевдокласса.

  Ниже примеры использования: 

A:link {color: blue} 
A:active {color: red}
A:visited {color; yellow}

  CSS определяет также псевдокласс для первой строки абзаца. Любое правило стиля, которое вы назначаете селектору по имени first-line, применяется к первой строке этого элемента. Если вы используете псевдокласс first-line, например, с дескриптором <P>, первая строка абзаца будет форматироваться согласно этому правилу. Следующий пример определяет вывод первой строки каждого абзаца полужирным шрифтом: 

P:first-line {font-weight: bold}

Точно так же вы можете изменять форматирование для первого символа в элементе, используя псеидокласс first-symbol. Это удобно для создания специальных эффектов выделения заглавных букв: 

Р:irst-letter {ont-size: 400%; float: left}


11. CSS Цвета


Цвета задаются с помощью предопределенного названия цветов или значения RGB, HEX, HSL, RGBA, HSLA.
В HTML, цвет может быть определен с помощью имени цвета, например как: 

<div style="background-color:tomato;">Tomato</div>

Некоторые примеры цветов:

Tomato
Orange
DodgerBlue
MediumSeaGreen
Gray
SlateBlue
Violet
LightGray

Вы можете установить цвет фона для HTML-элементов:

<div style="background-color:dodgerblue; text-align:center;color:white;font-size:24px;">

Привет, мир

Вы можете установить цвет текста:

<h3 style="color:tomato; text-align: center">
<p class="xxx" style="vertical-align: inherit;">Привет, мир</p>
</h3>

Привет, мир

В HTML цвета также могут быть определены с использованием значений RGB, HEX значения, значения HSL, значения RGBA и значения HSLA. То же значение цвета «Помидор»:

RGB (255, 99, 71)
# ff6347
HSL (9, 100%, 64%)

То же значение цвета «Томатный», но 50% прозрачно:

RGBA (255, 99, 71, 0,5)
HSLA (9, 100%, 64%, 0,5)

12. CSS обои


background-color Свойство задает цвет фона элемента. Цвет фона страницы устанавливается следующим образом: 

body {
background-color: lightblue;
}

background-image Свойство определяет изображение для использования в качестве фона элемента. По умолчанию изображение повторяется так оно охватывает весь элемент. Фоновое изображение для страницы можно установить следующим образом: 

body {
background-image: url("paper.gif");
}

По умолчанию background-image свойство повторяет изображение по горизонтали и по вертикали. Некоторые изображения должны повторяться только по горизонтали или по вертикали, или они будут выглядеть странно. Для повторений только по горизонтали background-repeat: repeat-x; , а для повторений изображения по вертикали, установите background-repeat: repeat-y;. background-repeat выставление повторяемости изображения, например: 

body {
background-image: url("img_tree.png");
background-repeat: no-repeat;
}

Положение изображения задается background-position свойством: 

body {
background-image: url("img_tree.png");
background-repeat: no-repeat;
background-position: right top;
}

Для того, чтобы указать , что фоновое изображение должно быть фиксированным (не будет прокручиваться с остальной частью страницы), используйте background-attachment свойство: 

body {
background-image: url("img_tree.png");
background-repeat: no-repeat;
background-position: right top;
background-attachment: fixed;
}

Для сокращения кода, также можно указать все свойства фона в одной собственности. Это называется сокращенным свойством. Сокращённое свойство для фона background: 

body {
background: #ffffff url("img_tree.png") no-repeat right top;
}

При использовании сокращённого свойства порядок значений свойств является:

  • background-color
  • background-image
  • background-repeat
  • background-attachment
  • background-position

Это не имеет значения, если одно из значений свойств отсутствует, при условии, что остальные находятся в таком порядке.
